With Fullui you are able to render child components using slots (like most libraries), but Fullui also introduces a fresh new way of rendering child components, which works great with content collections or any headless CMS.

:::note
This feature is currenty in beta.
If you find any problems or bugs report them here: [GitHub Issues](https://github.com/fullui/ui/issues).
:::

## Using Props
:::tip[Unique feature]
This feature allows you to build entire pages from data only. For example, from Astro collection or a headless CMS.
:::

Instead of the usual way of building components using slots (see below), in Fullui you can use props too. This unlucks the ability to dynamically generate entire pages from your Astro content collections for example.

Each component loops over it's props, rendering a child when:

- __key is `html`:__ the html html string is rendered
- __key ends with component name AND value is an object:__ the componet with name matching the end of the key is rendered
- __value is an object containing a `component` key:__ the specified component is rendered
- __value is an object containing a `as` key:__ the Tag component is renderer with the specified tag

``` astro live
---
import { Writeup } from '@fullui/ui'
---

<Writeup 
  align="start"
  subtext={{
    html: 'Lorem ipsum'
  }}
  heading={{
    html: 'Lorem ipsum dolor'
  }}
  text={{
    html: 'Lorem ipsum dolor sit amet, consectetur adipiscing elit, <br /> sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.'
  }}
  group={{
    button: {
      variant: 'surface',
      html: 'Lorem'
    },
    secondaryButton: {
      variant: 'ghost',
      html: 'Lorem'
    }
  }}
/>
```

#### Choosing component by name
You might want to choose you own props names, because it looks more neat in your cms for example.
``` astro live
---
import { Group } from '@fullui/ui'
---

<Group 
  button={{
    variant: 'surface',
    html: 'Lorem'
  }}
  foo={{
    component: 'Button',
    variant: 'surface',
    html: 'Lorem'
  }}
/>
```

#### Choosing the tag to be rendered
You can choose which HTML tag is used using `as` prop. This also signal to Fullui that this should be rendered as a isJsxChild, therefore the `component` prop or keyname is not needed anymore.
``` astro live
---
import { Heading } from '@fullui/ui'
---

<Heading
  html='Lorem ipsum '
  secondPart={{
    as: 'span',
    style: 'text-decoration:underline',
    html: 'dolor amet'
  }}
/>
```

## Using Slots
Or just can use the classic way of rendering children, using slots.

Each component has 3 slots:
- __default__: overwrites any children passed via props
- __before__: renders before the children passed via props
- __after__: renders after the children passed via props


``` astro live
---
import { Writeup, Subtext, Heading, Text, Group, Button } from '@fullui/ui'
---

<Writeup align="start">
  <Subtext>
    Lorem ipsum
  </Subtext>
  <Heading>
    Lorem ipsum dolor
  </Heading>
  <Text>
    Lorem ipsum dolor sit amet, consectetur adipiscing elit, <br />
    sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
  </Text>
  <Group>
    <Button variant="surface">
      Button
    </Button>
    <Button variant="ghost">
      Button
    </Button>
  </Group>
</Writeup>
```
